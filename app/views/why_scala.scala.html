Why use Scala?

It's about time!

Java has been around for many years, and has completely dominated a big part of the business. Despite of a sceptical reception in the early years, "too slow", "too much like C", it has proven to be a strong language together with the JVM, making it substantially easier to build big, performing and stable solution in a number of areas, including the enterprise business tier. The language has had it's natural evolution, with significant improvements on the way, but all in all, changes have come very slowly, especially compared with the speeds of the modern internet world. A new language is a natural consideration in this situation.

The Scalable language

The language is all about scalability - from small scripts and DSL's, to complex parser designs, to high performing server software, to DRY business code. The functional programming in Scala lends is very well suited to distributed systems and parallel execution without the dangers of locks.

The static language that feels dynamic1

[Scala] decreases the cost of typesafety AND increases the value!

Bill Venners, Feel of Scala
Even with a kick ass top of the class type-system, the generel piece of code will be very crisp: 
val map = Map("x",19)
will construct a Map with the types [String,Int] In most cases, the typeinformation in the code will roughly be equivalent to the information it carries, eg "x is a String". This is good for the fingers, easy on the eyes and it sharpens your programmers brain!

The static dynamic language

Scalas advanced type system supports structural types, which means that you can do some of the dynamic language specials like duck typing in a statically compiled way.

Here we define a function, that will accept any object regardless of its type, as long as it defines a method named 'close', that takes no parameters and returns Unit: 

                    def using[T <: { def close(): Unit }](object: T) ....
You could happily feed it JDBC-connections and File-handles, and the compiler will immediately complain, if you get it wrong.

As of Scala 2.10 you can even use the new feature dynamics which in effect will let you define an object, that the compiler will let you access in ways, that will not be checked until runtime. This might be reasonable, when you interact with other components, that will return dynamic data (eg JSON). Before you go crazy with this, please note alternatives, including future ones, involving something like Compiler Type Macros.

But why not choose Clojure/Groovy/Go/Ceylon/JRuby/Jython/Javascript/...

Well you should certainly have a go at other languages as well. Like travelling to foreign places, thou pleasant, will also often make you appreciate your home more. If your heart is with the dynamic languages and runtime types, that will cut down your choices, and you might not enjoy Scala as much. But if you liked Java and are looking for an upgrade, you must at the very least take Scala for a spin.
